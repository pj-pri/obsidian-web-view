# PRD — Obsidian Web Vault

**버전**: 1.0  
**작성일**: 2026-04-25  
**상태**: 초안

---

## 1. 배경 및 목적

현재 프로젝트는 로컬에서 `.md` 파일을 읽어 Obsidian 스타일로 렌더링하는 프론트엔드 프로토타입이다.
파일을 **읽는 것**만 가능하고, 생성·수정·삭제가 안 되며, 서버 배포 시 실사용이 불가능하다.

이 PRD는 **단일 사용자가 서버에 배포하여 실제로 노트를 관리**할 수 있는 수준으로 끌어올리는 것을 목표로 한다.

---

## 2. 목표 (Goals)

- `.md` 파일을 서버 파일시스템에 CRUD할 수 있다
- 어느 환경(VPS, NAS, 홈서버)에서도 `node server.js` 한 줄로 띄울 수 있다
- 노트 뷰어(렌더링·검색·그래프·백링크)가 실사용 수준으로 완성된다
- Phase 2에서 인증을 붙일 수 있도록 구조를 미리 준비한다

## 3. 비목표 (Non-goals)

- 실시간 협업, 멀티유저 (→ 범위 외)
- 인증·접근 제어 (→ Phase 2로 이관)
- 리치 에디터(CodeMirror 등) (→ Phase 3로 이관)
- 모바일 앱
- Obsidian 플러그인 호환

---

## 4. 현재 상태

| 항목 | 상태 |
|------|------|
| 파일 읽기 (`GET /api/vault`) | 구현됨 — 전체 파일을 한 번에 로드 |
| 파일 쓰기 / 생성 / 삭제 | 없음 |
| 에디터 변경사항 저장 | localStorage에만 저장됨 |
| 검색 | 클라이언트 측 필터링만 |
| 서버 배포 | 가능하나 쓰기 API 없어 실사용 불가 |
| 이미지·첨부파일 | 없음 |
| 인증 | 없음 |

---

## 5. 사용자 시나리오

> 단일 사용자, 서버(VPS/NAS)에 배포, 브라우저로 접근

1. **노트 작성**: 사이드바에서 "+" 클릭 → 제목 입력 → 에디터에서 작성 → 자동 저장
2. **노트 수정**: 파일 클릭 → 에디터에서 편집 → 저장
3. **노트 삭제**: 파일 우클릭 → 삭제 확인
4. **파일 이동/이름 변경**: 드래그 또는 우클릭 → 이름 변경
5. **검색**: 사이드바 검색창 → 파일명·본문 전체 검색
6. **이미지 삽입**: 에디터에서 이미지 붙여넣기 또는 드래그 → 서버 업로드 → 마크다운 링크 삽입
7. **그래프 탐색**: 그래프 뷰에서 노드 클릭 → 해당 노트 열기

## 6. 기능 요구사항

### 6.1 백엔드 API

현재 `/api/vault` 단일 엔드포인트에서 **전체 파일 내용을 한 번에** 내려주는 구조는 vault가 커지면 느려진다.
파일 목록(메타데이터)과 파일 내용을 분리하여 **온디맨드 로딩**으로 전환한다.

#### 파일 API

| 메서드 | 경로 | 설명 |
|--------|------|------|
| `GET` | `/api/vault/files` | 파일 목록 (경로·수정일·크기만, 내용 제외) |
| `GET` | `/api/vault/files/:path` | 파일 내용 단건 조회 |
| `POST` | `/api/vault/files/:path` | 파일 생성 (body: `{ content }`) |
| `PUT` | `/api/vault/files/:path` | 파일 내용 전체 갱신 |
| `PATCH` | `/api/vault/files/:path` | 파일 이동·이름 변경 (body: `{ newPath }`) |
| `DELETE` | `/api/vault/files/:path` | 파일 삭제 |

#### 검색 API

| 메서드 | 경로 | 설명 |
|--------|------|------|
| `GET` | `/api/vault/search?q=xxx` | 파일명·본문 전체 텍스트 검색, 결과에 매칭 컨텍스트 포함 |

#### 첨부파일 API

| 메서드 | 경로 | 설명 |
|--------|------|------|
| `POST` | `/api/vault/attachments` | 이미지·파일 업로드 (multipart/form-data) |
| `GET` | `/api/vault/attachments/:filename` | 첨부파일 서빙 |

#### 공통 요구사항

- 모든 응답은 JSON (에러 포함)
- 경로 트래버설 방지 (`../` 차단) — 현재 `safeResolve` 로직 재활용
- `OBSIDIAN_VAULT_DIR` 환경변수로 vault 루트 지정 (기존 방식 유지)
- Phase 2 인증 대비: 요청 파이프라인에 미들웨어 슬롯 예약 (지금은 no-op)

### 6.2 프론트엔드

#### 6.2.1 파일 로딩 방식 전환

- 앱 초기화 시 `GET /api/vault/files`로 **목록만** 가져온다
- 파일 열기 시 `GET /api/vault/files/:path`로 **내용을 온디맨드** 로드한다
- 로드된 파일 내용은 인메모리 캐시에 저장한다

#### 6.2.2 저장 흐름

- 에디터 변경 시 **자동 저장** (debounce 1~2초), `PUT /api/vault/files/:path` 호출
- 서버 연결 없을 때(오프라인 fallback)는 기존처럼 localStorage 유지
- 저장 상태 표시: 상태바에 "저장 중…" / "저장됨" / "오류" 인디케이터

#### 6.2.3 파일 관리 UI

- 사이드바 파일 우클릭 메뉴: **이름 변경**, **삭제**, **경로 복사**
- 사이드바 폴더 우클릭 메뉴: **새 노트**, **새 폴더**, **삭제**
- 삭제 시 확인 다이얼로그 필수
- 파일 이름 변경 시 해당 파일을 `[[wikilink]]`로 참조하는 모든 노트의 링크 자동 업데이트

#### 6.2.4 검색 개선

- 현재 클라이언트 측 필터링 → `GET /api/vault/search` API 호출로 전환
- 검색 결과에 **본문 컨텍스트(매칭 라인)** 표시
- 검색 결과 클릭 시 해당 노트 열기 + 매칭 위치로 스크롤

#### 6.2.5 이미지·첨부파일

- 에디터에서 이미지 **붙여넣기(Paste)** 또는 **드래그앤드롭** 시 자동 업로드
- 업로드 완료 후 `![filename](attachments/filename.png)` 마크다운으로 삽입
- 프리뷰에서 `/api/vault/attachments/:filename` 경로로 렌더링

#### 6.2.6 뷰어 완성도

- **Frontmatter** YAML 파싱 및 표시 (제목·태그·날짜 등)
- **목차(TOC)** 자동 생성 (H2·H3 기준, 우측 패널 또는 프리뷰 상단)
- **코드 블록** 언어별 구문 강조 (highlight.js, CDN)
- **테이블** 렌더링 개선
- **체크박스** 클릭 시 원본 markdown 토글 반영
- 깨진 `[[wikilink]]` 시각적 구분 (현재 구현됨, 유지)

## 7. 기술 아키텍처

```
브라우저
  └─ React (CDN, 빌드 없음)
       ├─ app.jsx        — UI 컴포넌트, 상태 관리
       ├─ markdown.js    — 마크다운 파서
       └─ graph.jsx      — 그래프 뷰

Node.js 서버 (server.js)
  ├─ 정적 파일 서빙     — HTML/JS/CSS
  └─ vault-api/
       ├─ router.js      — URL 라우팅 (신규)
       ├─ files.js       — 파일 CRUD 핸들러 (신규)
       ├─ search.js      — 검색 핸들러 (신규)
       ├─ attachments.js — 첨부파일 핸들러 (신규)
       └─ index.js       — 기존 진입점 (리팩토링)

파일시스템
  └─ $OBSIDIAN_VAULT_DIR/
       ├─ *.md
       ├─ folder/
       │    └─ *.md
       └─ attachments/
            └─ *.png, *.jpg, ...
```

### 의존성 정책

- **Node.js 기본 모듈만** 사용 (npm 의존성 없음 유지)
- 예외: 첨부파일 multipart 파싱은 직접 구현하거나 경량 라이브러리 1개 허용 검토

## 8. 릴리즈 단계

### Phase 1 — 실사용 가능한 CRUD (현재 목표)

**백엔드**
- [ ] 파일 목록 API (`GET /api/vault/files`)
- [ ] 파일 단건 조회 API (`GET /api/vault/files/:path`)
- [ ] 파일 생성 API (`POST /api/vault/files/:path`)
- [ ] 파일 수정 API (`PUT /api/vault/files/:path`)
- [ ] 파일 삭제 API (`DELETE /api/vault/files/:path`)
- [ ] 파일 이동/이름 변경 API (`PATCH /api/vault/files/:path`)
- [ ] 전체 텍스트 검색 API (`GET /api/vault/search`)
- [ ] 첨부파일 업로드/서빙 API

**프론트엔드**
- [ ] 온디맨드 파일 로딩으로 전환
- [ ] 자동 저장 (debounce + 저장 상태 인디케이터)
- [ ] 파일/폴더 우클릭 컨텍스트 메뉴
- [ ] 검색 API 연동
- [ ] 이미지 붙여넣기·드래그 업로드

**인프라**
- [ ] `Dockerfile` 작성 (단일 컨테이너)
- [ ] 환경변수 목록 문서화 (`.env.example`)

### Phase 2 — 인증 & 보안

- [ ] HTTP Basic Auth 또는 토큰 기반 인증 미들웨어
- [ ] HTTPS 설정 가이드 (reverse proxy: nginx/caddy)
- [ ] Rate limiting

### Phase 3 — 에디터 업그레이드

- [ ] CodeMirror 6 (또는 Milkdown) 통합
- [ ] 실시간 위지윅(WYSIWYG) 마크다운
- [ ] 자동완성: `[[`, `#tag`, `/snippet`

## 9. 성공 지표

| 지표 | 기준 |
|------|------|
| 노트 저장 | 편집 후 2초 내 디스크에 반영 |
| 파일 열기 속도 | 500KB 미만 파일 300ms 내 로드 |
| 검색 응답 | 1,000개 파일 기준 500ms 내 결과 반환 |
| 서버 배포 | Docker 이미지 빌드·실행 5분 내 완료 |
| 안정성 | 경로 트래버설 등 보안 입력으로 500 에러 없음 |
